# style-dataset-lab

Comfy-backed visual dataset factory. Generate, curate, and export multimodal training data for vision-language model fine-tuning.

## What this is

A repo whose whole job is to become **trainable visual truth**. Every asset carries three things:

1. **Image pixels** — generated by ComfyUI with full provenance (checkpoint, LoRA, seed, sampler, cfg)
2. **Canon explanation** — why this is on-style or off-style, grounded in a style constitution
3. **Quality judgment** — approved/rejected with per-dimension scores and cited rules

The output feeds into [`@mcptoolshop/repo-dataset`](https://github.com/mcp-tool-shop-org/repo-dataset) to produce multimodal training data in TRL, LLaVA, Qwen2-VL, Axolotl, and LLaMA-Factory formats.

## Current lane: Gritty Space

Uniforms and costume design for a lived-in sci-fi universe. Grounded in a visual design bible with 5 faction DNA cards, 3 alien species anatomy specs, clothing layer systems, and rank progressions.

**Art pillars:** Worn, Functional, Tense, Culturally specific, Scrappy.

**Style reference:** Firefly wardrobes, Alien (1979) industrial grit, Cowboy Bebop crew quarters — working-class space, not soldier-class space.

## Workflow

```bash
# 1. Start ComfyUI
cd F:/AI-Models/ComfyUI-runtime && python main.py --listen 127.0.0.1 --port 8188

# 2. Generate candidates from a prompt pack
node scripts/generate.js inputs/prompts/wave1.json
node scripts/generate.js inputs/prompts/wave1.json --dry-run

# 3. Curate — approve, reject, or mark borderline
node scripts/curate.js <asset_id> approved "explanation" --scores "silhouette:0.9,palette:0.8"
node scripts/curate.js <asset_id> rejected "explanation" --failures "too_clean,wrong_material"
node scripts/curate.js --list

# 4. Record pairwise comparisons
node scripts/compare.js <asset_a> <asset_b> a "A has better faction read because..."

# 5. Export training data via repo-dataset
node F:/AI/repo-dataset/dist/cli.js visual generate . --format trl --output exports --allow-incomplete
node F:/AI/repo-dataset/dist/cli.js visual inspect .
node F:/AI/repo-dataset/dist/cli.js visual validate exports/dataset.jsonl
```

## Generation setup

```yaml
checkpoint: dreamshaperXL_v21TurboDPMSDE.safetensors
lora: classipeintxl_v21.safetensors (weight: 1.0)
resolution: 1024x1024
steps: 8
cfg: 2.0
sampler: dpmpp_sde
scheduler: karras
speed: ~9s per image (RTX 5080)
```

## Directory structure

```
canon/              Style constitution + review rubric (13 rules, 8 scoring dimensions)
inputs/prompts/     Prompt packs per wave (JSON: subjects, variations, defaults)
inputs/references/  IP-Adapter reference images
inputs/control/     ControlNet control images
outputs/candidates/ Raw generations (gitignored)
outputs/approved/   Curated approved (gitignored)
outputs/rejected/   Curated rejected (gitignored)
outputs/borderline/ Curated borderline (gitignored)
records/            Per-asset JSON (provenance + judgment + canon binding)
comparisons/        A-vs-B preference judgments
exports/            repo-dataset output (gitignored)
scripts/            generate.js, curate.js, compare.js, bulk-curate
```

## Requirements

- [ComfyUI](https://github.com/comfyanonymous/ComfyUI) running on localhost:8188
- DreamShaper XL Turbo checkpoint + ClassipeintXL LoRA
- Node.js 20+
- [`@mcptoolshop/repo-dataset`](https://github.com/mcp-tool-shop-org/repo-dataset) for export
