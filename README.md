# style-dataset-lab

Visual dataset factory -- generate, curate, and export multimodal training data for VLM fine-tuning.

## What this is

A **monorepo** toolkit for building **trainable visual truth** across multiple games. Each game gets its own data directory; scripts are shared. Every asset carries three things:

1. **Image pixels** -- generated by ComfyUI with full provenance (checkpoint, LoRA, seed, sampler, cfg)
2. **Canon explanation** -- why this is on-style or off-style, grounded in a style constitution
3. **Quality judgment** -- approved/rejected with per-dimension scores and cited rules

The output feeds into [`@mcptoolshop/repo-dataset`](https://github.com/mcp-tool-shop-org/repo-dataset) to produce multimodal training data in 10 formats: TRL, LLaVA, Qwen2-VL, Axolotl, LLaMA-Factory, ShareGPT, RLHF pairs, DPO, captioning, and classification.

## Security model

**Local-only.** style-dataset-lab talks to ComfyUI on `localhost:8188` and never makes external network requests. No telemetry, no analytics, no phoning home. Image generation happens entirely on your GPU. Records and canon data stay on your filesystem.

## Dataset stats (Star Freight)

The included `games/star-freight/` dataset ships with the repo as a working example:

| Metric | Value |
|--------|-------|

| Distinct rejection types | 30+ |
| Identity packet system | Named characters with faction DNA, rank, and visual anchors |
| Export formats | 10 (via @mcptoolshop/repo-dataset) |

## Install

```bash
# Install scripts globally (optional)
npm install -g @mcptoolshop/style-dataset-lab

# Clone the repo (includes star-freight game data)
git clone https://github.com/mcp-tool-shop-org/style-dataset-lab
cd style-dataset-lab
npm install
```

To start a new game, create its directory structure:

```bash
mkdir -p games/my-game/{canon,records,comparisons,inputs/prompts,outputs/{candidates,approved,rejected},exports}
```

Or use the existing `games/star-freight/` data to explore the workflow immediately.

## Monorepo structure

Each game lives in `games/<name>/` with its own canon, records, and assets. Scripts are shared and accept `--game <name>` (defaults to `star-freight`).

```
style-dataset-lab/
  scripts/                  Shared tooling (generate, curate, compare, etc.)
  games/
    star-freight/           Star Freight universe data
      canon/                Style constitution, review rubric, species canon
      records/              Per-asset JSON (provenance + judgment + canon)
      comparisons/          A-vs-B preference judgments
      inputs/               Prompt packs, identity packets, references
      outputs/              Generated images (gitignored)
      exports/              repo-dataset output (gitignored)
    fractured-road/         (add your next game here)
      canon/
      records/
      ...
```

## Workflow

```bash
# 1. Start ComfyUI
# (point it at your checkpoint + LoRA setup)

# 2. Generate candidates from a prompt pack
npm run generate -- --game star-freight inputs/prompts/wave1.json
npm run generate -- --game star-freight inputs/prompts/wave1.json --dry-run

# 3. Generate identity-packet characters
npm run generate:identity -- --game star-freight inputs/identity-packets/wave27a.json

# 4. Generate painterly variants of approved assets
npm run painterly -- --game star-freight

# 5. Curate -- approve, reject, or mark borderline
npm run curate -- --game star-freight <asset_id> approved "explanation"
npm run curate -- --game star-freight <asset_id> rejected "explanation" --failures "too_clean"

# 6. Bind canon explanations to assets
npm run canon-bind -- --game star-freight

# 7. Record pairwise comparisons
npm run compare -- --game star-freight <asset_a> <asset_b> a "A has better faction read"

# 8. Export training data via repo-dataset
repo-dataset visual generate ./games/star-freight --format trl
repo-dataset visual inspect ./games/star-freight
```

## Adding a new game

```bash
mkdir -p games/my-game/{canon,records,comparisons,inputs/prompts,outputs/{candidates,approved,rejected},exports}
# Write your canon/constitution.md and canon/review-rubric.md
# Create prompt packs in inputs/prompts/
# Then generate, curate, and export as above with --game my-game
```

## Per-game directory layout

Each `games/<name>/` directory contains:

```
canon/                  Style constitution, review rubric, species canon, identity gates
inputs/
  prompts/              Prompt packs per wave (JSON: subjects, variations, defaults)
  references/           IP-Adapter reference images
  control-guides/       ControlNet guide overlays
  identity-packets/     Named character identity spines
outputs/
  candidates/           Raw generations (gitignored)
  approved/             Curated approved (gitignored)
  rejected/             Curated rejected (gitignored)
  borderline/           Curated borderline (gitignored)
  painterly/            Painterly-style variants (gitignored)
records/                Per-asset JSON (provenance + judgment + canon binding)
comparisons/            A-vs-B preference judgments
exports/                repo-dataset output (gitignored)
```

## Generation setup

```yaml
checkpoint: dreamshaperXL_v21TurboDPMSDE.safetensors
lora: classipeintxl_v21.safetensors (weight: 1.0)
resolution: 1024x1024
steps: 8
cfg: 2.0
sampler: dpmpp_sde
scheduler: karras
speed: ~9s per image (RTX 5080)
```

Additional generation modes: ControlNet (pose/depth-guided), IP-Adapter (reference-driven), and identity packets (named character consistency).

## Requirements

- [ComfyUI](https://github.com/comfyanonymous/ComfyUI) running on localhost:8188
- DreamShaper XL Turbo checkpoint + ClassipeintXL LoRA
- Node.js 20+
- [`@mcptoolshop/repo-dataset`](https://github.com/mcp-tool-shop-org/repo-dataset) for training export
