Style Dataset Lab is a **production pipeline** for visual dataset creation. It takes you from style rules (canon) through structured prompts to ComfyUI generation, human curation, canon binding, and training export -- across multiple games, each with its own data directory under `games/<name>/`.

The pipeline ships as an npm package (`@mcptoolshop/style-dataset-lab`) with 13 scripts and blank templates. Install it, write your canon, and start producing datasets. The repo also includes `games/star-freight/` as a working example with 1,182 records.

## The pipeline

```
Write Canon (constitution + rubric)
    |
    v
Create Prompt Packs (.json)
    |
    v
generate.js --> ComfyUI HTTP API --> candidate images
    |
    v
curate.js --> per-dimension scoring --> approved / rejected / borderline
    |
    v
painterly.js --> img2img style pass --> painterly variants
    |
    v
canon-bind.js --> constitution rules --> pass/fail/partial assertions per rule
    |
    v
compare.js --> pairwise A-vs-B --> preference pairs
    |
    v
repo-dataset --> TRL / LLaVA / Qwen2-VL / Axolotl / LLaMA-Factory
```

Each stage writes structured JSON records to `games/<name>/records/`. All scripts accept `--game <name>` (defaults to `star-freight`). Records accumulate provenance, judgment, and canon binding over time. Nothing is lost -- a curated record still carries its original generation provenance.

## The training triangle

Every record in the dataset carries three things:

1. **Image pixels** -- generated by ComfyUI with full provenance (checkpoint, LoRA, seed, sampler, cfg scale, resolution).
2. **Canon explanation** -- why this image is on-style or off-style, grounded in a style constitution with named rules.
3. **Quality judgment** -- approved, rejected, or borderline, with per-dimension scores and cited failure modes.

This triple -- image, canon, judgment -- is what makes the dataset useful for fine-tuning. A model trained on images alone learns to generate. A model trained on images with grounded judgment learns to *evaluate*.

## What it produces

The export pipeline (via `@mcptoolshop/repo-dataset`) reads the records and images to produce multimodal training data in multiple formats:

| Training type | What it teaches |
|---------------|-----------------|
| **Classification** | Given an image, predict approved/rejected/borderline |
| **Critique** | Given an image, explain what is on-style and what fails |
| **Preference** | Given two images, pick the better one and explain why |

## Multi-game structure

The pipeline is game-agnostic. Each game lives in `games/<name>/` with its own canon, records, comparisons, inputs, outputs, and exports. The 13 scripts in `scripts/` accept `--game <name>` to target any game. Blank templates in `templates/` give you a starting point.

To add a new game from templates:

```bash
mkdir -p games/my-game/{records,comparisons,inputs/prompts,outputs/{candidates,approved,rejected,borderline,painterly},exports}
cp -r templates/canon games/my-game/canon
cp templates/inputs/prompts/example-wave.json games/my-game/inputs/prompts/wave1.json
```

## Example dataset: Star Freight

The repo includes `games/star-freight/` as a working example -- a lived-in sci-fi universe with 5 factions, 1,182 curated records, 28 prompt waves, and 18 visual categories. Clone the repo to explore it.
